### 1. 리눅스 명령어 (top, ps, jobs, kill)
#### 1) **top 명령어**  
  **top** 명령어는 **현재** OS상태를 나타내어주는 **CLI 어플리케이션임!**  
    + CPU, RAM, PROCESS등 시스템의 상태를 빠르게 알아볼 수 있음.  
    + **TOP**명령어   

        Shift + p : CPU사용율 내림차순 정렬  
        Shift + m : 메모리 사용율 내림차순 정렬  
        Shift + t : 프로세스가 돌아가는 시간순서로 정렬  
        k         : kill, k 입력후 PID번호 작성. signal은 9임  
        f         : sort field 선택 화면 -> q 누르면 RES순으로 정렬  
        a         : 메모리 사용량에 따라 정렬  
        b         : Batch 모드로 작동함  
        1         : CPU 코어별로 사용량을 보여줌  
        (PID: 프로세스 ID...?)  
        (RES: 누수 체크가 가능한 메모리 메모리의 사용량을 보여줌.
        또한, RES는 현재 프로세서가 사용하고 있는 물리 메모리의 양을 표시해주므로 중요함!)

#### 2) **ps 명령어**
  **ps** 명령어는 위의 **top** 명령어와 매우 비슷함!
    + 그렇다면 왜 ps명령어가 따로 있는가?  
      top는 **현재**의 시각에 따른 사용률을 보여준다고 가정을 한다면, ps는 **현재가 아닌 프로세스 전체 사용시간의 사용률**을 보여준다!  
    + **PS**명령어  
    
         ps -e   : 커널 프로세서를 제외한 모든 프로세스 출력  
         ps -f   : 포멧 형태로 출력(풀 형태)  
         ps -l   : 포멧 형태로 출력(긴 형태)  
         ps -A   : 모든 프로세서 출력  
         ps -a   : 세션리더를 제외하고 종속되지 않은 모든 프로세서 출력  
         ps  a   : 터미널과 관련된 프로세서를 출력하는 옵션  
         ps -p   : 특정 PID를 지정할 때 사용되는 옵션  
        
#### 3) **jobs 명령어**
  **jobs** 명령어는 작업의 상태를 표시하는 명령어로, 쉘 세션에서 실행하고 있는 백그라운드의 작업 목록을 확인할 수 있다!  
  각 작업에는 번호가 붙어있어 kill명령어를 사용하기에 더 간편하다!  
    + **jobs**명령어     
         
         -l      : 프로세서 그룹 ID를 state 필드 앞에 출력
         -n      : 프로세서 그룹 중에 대표 프로세스 ID를 출력
         -p      : 각 프로세서 ID에 대해 한 행씩 출력
         command : 지정한 명령어를 실행
         
#### 4) **kill 명령어**
  **kill** 명령어는 리눅스 작업중 프로세스가 작동을 하지 않을때, 종료를 하기 위한 명령어이다!  
   안전하게 종료(방법은 SIGTERM 시그널을 보냄. 기본적인 동작 자체가 프로그램 종료)를 진행해줌!**절때 강제 종료가 아님!**  
   강제 종료하는 방법은 SIGKILL 시그널을 보내면 됨. (권장X)  
    + 사용 방법  
      **kill Options PID**  
      Options에는 명령어 번호를 ex) -9, -15  
      아니면 이름을 ex) Sigkill, Sigterm 사용해도 상관이 없다.  
    + **kill**명령어  
    
           kill 명령어는 매우 다양하고 종류가 많다... 주로 사용하는 2개를 가져온다면.....
           Signal 종류 9번, 15번을 자주 사용한다.
           9 ) Sigkill : 죽이기 (강제 종료라고 생각하면 됨.)
           15) Sigterm : 소프트웨어 종료 시그널
